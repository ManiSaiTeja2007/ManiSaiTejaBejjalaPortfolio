Embedded System Monitor (C)

A real-time monitoring system.

Image of the embedded system hardware setup.

Detailed Description

This project focused on developing a real-time monitoring system using a microcontroller (e.g., STM32, ESP32) programmed in C. The system is designed to collect data from various sensors (temperature, humidity, light) and display it on an LCD or send it to a serial terminal for analysis.

  • Problem Solved: Created a compact and efficient solution for localized environmental monitoring, applicable in various contexts like greenhouses, data centers, or smart homes.
  • My Role: I was responsible for selecting the microcontroller, designing the sensor interface circuits, writing the low-level C drivers for the sensors and display, and implementing the real-time data acquisition and processing logic.
  • Key Features: Multi-sensor data acquisition, real-time data display (LCD), serial communication for debugging/logging, configurable sampling rates.
  • Technologies Used: C programming language, specific microcontroller (e.g., STM32F4 series, ESP32), various sensors (DHT11/BMP180, LDR), I2C/SPI communication protocols, embedded development environment (e.g., PlatformIO, Arduino IDE for ESP32).
  • Challenges and Solutions: Accurately reading and calibrating sensor data was a challenge, especially for analog sensors. This was addressed by implementing averaging filters and carefully following sensor datasheets for calibration procedures. Managing memory and processing power constraints on the microcontroller also required efficient C code.
  • Learnings: Deepened my understanding of embedded C programming, real-time operating systems (RTOS concepts if applicable), sensor integration, and optimization for constrained hardware environments.

Technologies Used

C Embedded Systems Microcontrollers (e.g., STM32/ESP32) Sensors I2C/SPI
